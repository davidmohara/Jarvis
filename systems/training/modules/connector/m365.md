## What You'll Learn

What the Microsoft 365 connector enables across all agents — email context, calendar data, Teams messages, and how each agent uses this data differently.

## Before We Start

M365 connector must be active in the user's config.

## Walkthrough

### Step 1: What M365 Provides (3 min)

Walk through the data layer:

- **Email** — Recent emails, threads, sent/received, attachments
- **Calendar** — Events, attendees, scheduling, free/busy
- **Teams** — Chat messages, channel posts, meeting transcripts

### Step 2: How Each Agent Uses It (5 min)

**Chief** uses email and calendar for:
- Morning briefing (today's meetings, key emails)
- Inbox processing (email triage)
- Calendar prep (attendee context from email threads)

**Chase** uses email and calendar for:
- Client interaction history (when did we last email/meet?)
- Deal activity tracking (proposals sent, meetings held)

**Shep** uses email, calendar, and Teams for:
- 1:1 prep (interaction recency, conversation threads)
- Team health pulse (interaction patterns, response times)

**Harper** uses email for:
- Email drafting (threading, context from previous messages)
- Communication patterns (what tone works with which contacts)

**Quinn** uses calendar for:
- Alignment checks (where is time being spent?)
- Initiative tracking (meeting cadence on strategic projects)

**Coaching prompt:** "One connector feeds all five agents. That's the leverage. You connect once, and every briefing, every prep, every review gets richer."

### Step 3: Privacy and Scope (2 min)

Explain what the system can and cannot access:
- Only accesses data the user has permission to see
- Does not store email content in IES files (references, not copies)
- Can be disconnected at any time

## Reflection

Ask: "Now that you see how M365 data flows through the system, which agent's output improved the most?"

## Success Criteria

Understand what M365 connection enables across agents. The user can name at least 3 specific use cases.
