## What You'll Learn

The 5 operations you'll use every day: `/capture`, `/find`, `/archive`, `/status`, and `/prioritize`. These are the verbs of the system.

## Before We Start

Complete the System Orientation module first (or at least know the basics).

## Walkthrough

### Operation 1: `/capture` (3 min)

The fastest operation. Something comes to mind — capture it before it evaporates.

**Live demo:** Ask the user what's on their mind right now that they haven't written down. Run `/capture` with their answer. Show them the task appearing in their inbox.

**Coaching prompt:** "This is the escape valve. In a meeting and someone says something you need to follow up on? Just capture it. You'll process it later. The point is: nothing gets lost."

### Operation 2: `/find` (3 min)

Search across the system — files, tasks, knowledge, context.

**Live demo:** Ask the user to name a project or person they work with frequently. Run `/find` with that term. Show them what surfaces — files, delegations, objectives, notes.

**Coaching prompt:** "You don't need to remember where things are filed. Just ask. The system searches everywhere — files, task management, knowledge base, objectives."

### Operation 3: `/status` (3 min)

Quick pulse check. Where am I? What's pending?

**Live demo:** Run `/status` now. Walk through the output — rocks progress, overdue items, inbox count, upcoming events.

**Coaching prompt:** "This is your 30-second gut check. If you only have time for one thing between meetings, run status. It tells you if anything's on fire."

### Operation 4: `/prioritize` (3 min)

When the list is long and everything feels urgent.

**Live demo:** If the user has tasks in their inbox or overdue items, run `/prioritize` and walk through how the system ranks them — using the Eisenhower matrix (urgent/important) and their quarterly rocks for alignment.

**Coaching prompt:** "Prioritize doesn't just sort by due date. It looks at what's tied to your rocks, what has dependencies, and what's been sitting too long. It's a thinking tool, not just a sorting tool."

### Operation 5: `/archive` (3 min)

Close completed work cleanly. Move it out of active view.

**Live demo:** If there's any completed item in the system, archive it. Show how it moves to the archive directory but remains searchable via `/find`.

**Coaching prompt:** "Archive aggressively. If it's done, move it. A clean active workspace means you're only looking at what matters. You can always find archived items later."

## Reflection

Ask: "Which of these five do you think you'll use most? And which one do you think you should use most but probably won't?"

The gap between those two answers is a coaching opportunity.

## Success Criteria

Use all 5 operations independently in a single session. Mastered after doing this twice.
