## What You'll Learn

How to start your day with Chief — the morning briefing that surfaces priorities, calendar context, overdue items, and key decisions for today. This is the keystone habit of IES.

## Before We Start

No prerequisites. This is the first agent module everyone learns.

If connectors are active (email, calendar), the briefing is richer. If not, Chief works with what's available and the briefing improves as data flows in.

## Walkthrough

### Step 1: Trigger the Briefing (2 min)

Say "start my day" or run `/boot`.

**Coaching prompt:** "This is what you'll do every morning. Just open the system and say 'start my day.' Chief handles the rest."

Trigger the `chief-morning` skill. Let it run fully.

### Step 2: Read the Calendar Block (3 min)

Walk through today's calendar:

- Point out how many meetings are scheduled
- Identify any that overlap or are back-to-back
- Highlight the one meeting that needs the most preparation

**Coaching prompt:** "Chief pulls your live calendar. If a meeting got added or cancelled since yesterday, it shows here — not some stale file from last week."

For role-specific coaching:
- **CEO**: "Which of these meetings is a decision meeting vs. an update? You should only attend decision meetings."
- **VP**: "Any client meetings? Those get prep automatically — we'll learn that in a later module."
- **COS**: "Look for conflicts across the leadership team's calendars. That's your superpower."

### Step 3: Read the Priority Stack (3 min)

Walk through the priority section:

- Overdue delegations (if any)
- Tasks due today
- Items from yesterday's daily review that carried forward

**Coaching prompt:** "Chief orders these by impact, not just by due date. Overdue delegations always come first because those are broken commitments to other people."

### Step 4: Identify the One Thing (3 min)

Ask the user: "Looking at this briefing, what's the one thing you need to handle before anything else today?"

Listen to their answer. If it aligns with the priority stack, reinforce: "That's what Chief flagged too."

If it doesn't align, explore: "Interesting — Chief put [X] at the top. What makes [their choice] more important?" This builds the user's muscle for challenging the system's recommendations thoughtfully.

### Step 5: Take One Action (4 min)

Don't just read the briefing — act on it. Pick one item from the briefing and handle it right now:

- Overdue delegation → draft a follow-up message
- Meeting prep needed → run a quick prep
- Inbox item → capture or process it
- Task due → check if it's actually done

**Coaching prompt:** "The morning briefing isn't a report to read — it's a launchpad. The best practice is to handle the top priority item immediately, before email or meetings pull you off track."

## Reflection

Ask: "Was there anything in the briefing that surprised you — something you'd forgotten about or didn't expect?"

This builds awareness of what the system catches that human memory doesn't.

## Success Criteria

Run the morning briefing 3 times independently. By the third time, the user should be:
- Scanning the briefing in under 2 minutes
- Identifying their top priority without prompting
- Taking one action before moving on
