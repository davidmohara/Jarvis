## What You'll Learn

How to build an evidence-based performance review using data from the system — delegation history, interaction patterns, goal achievement, and coaching notes.

## Before We Start

This module requires active history — delegations completed/overdue, 1:1 notes, coaching observations. Best done after the user has been using the delegation and 1:1 modules for several weeks. If insufficient data exists, Shep should note: "The review will be richer once we have more history. Let's start with what we have."

## Walkthrough

### Step 1: Pick the Person (3 min)

Who needs a performance review? Ask the user to select someone and state the purpose — annual review, mid-year check-in, promotion consideration, or performance improvement.

### Step 2: Gather the Evidence (7 min)

Pull all available data for this person:

- Delegation completion rate (on-time, late, quality)
- 1:1 notes and coaching themes over time
- Interaction frequency and recency
- Goals and milestones assigned to them
- Any notes or observations from the knowledge base

**Coaching prompt:** "Performance reviews should be evidence-based, not impression-based. What the system gives you is the facts. Your job is to interpret them with context."

### Step 3: Structure the Review (10 min)

Build the review around four areas:

**Results** — What did they deliver? Match against goals and delegations.
**Behavior** — How did they work? Patterns from interactions, responsiveness, collaboration.
**Growth** — What's changed since last review? Skills developed, responsibilities expanded.
**Potential** — What's next? Where could they grow, what's holding them back?

For each area, work with the user to identify 1-2 specific examples. "Don't say 'she's reliable.' Say 'she delivered the Q1 pipeline report on time every week despite the CRM migration.'"

### Step 4: The Hard Conversation (5 min)

**Coaching prompt:** "Every performance review has at least one difficult truth. What's the thing you know you need to say but are tempted to skip?"

Help the user articulate the constructive feedback clearly:
- State the observation (not interpretation)
- State the impact (who/what was affected)
- State the expectation (what good looks like)
- Ask for their perspective (it's a dialogue, not a verdict)

### Step 5: Save and Prepare (5 min)

Save the review prep to the knowledge base. Set a reminder for post-review follow-up (to capture the person's response and agreed action items).

## Reflection

Ask: "Is there anything about this person's performance that surprised you when you saw the data?"

The gap between impression and evidence is the coaching opportunity.

## Success Criteria

Build one performance review with evidence from interactions, delegations, and goals. The review should include specific examples, not just general impressions.
