Deploy safely or don't deploy at all.## INITIALIZATION### Input Required- `evolution_path` — Path to evolution package directory containing evolution.manifest.json### Data Sources Required| Source | What to Pull | Access Method ||--------|-------------|---------------|| Evolution manifest | Package metadata, file list, actions | Read evolution_path/evolution.manifest.json || Current system files | Agent definitions, workflows, permissions | File system reads || Evolution history | Previously applied evolutions | Read evolutions/history.md || System snapshots | Available rollback points | List evolutions/snapshots/ directory |### Pre-Flight RequirementsBefore starting workflow:1. Verify evolution_path exists and contains evolution.manifest.json2. Confirm current working directory is project root3. Check write permissions to all target directories4. Verify git status is clean (recommended)## EXECUTION### Step SequenceThis workflow MUST be executed in order. Each step has defined outputs that feed the next step. Do not skip steps. Do not proceed if a step fails.1. **Step 01: Validate Manifest** — Parse and validate evolution manifest structure2. **Step 02: Compatibility Check** — Verify minimum_base_version requirement3. **Step 03: Create Snapshot** — Backup current state of all files4. **Step 04: Scan Personal Blocks** — Parse target files, extract personal block locations5. **Step 05: Apply Evolution** — Execute file operations (add/replace/merge/delete)6. **Step 06: Verify & Log** — Confirm integrity, write to history### Execution Entry PointRead and follow: `steps/step-01-validate-manifest.md` to begin the workflow.## CONFLICT HANDLINGIf any step detects a conflict:1. HALT immediately — do not proceed2. Surface the conflict with file path, action type, personal blocks affected3. Offer options: Apply and preserve, Skip file, Abort, Review manually4.
