# Agent Behavior Preferences

David's preferences for how IES agents behave. Updated by David or Rigby on request.

## Communication Style
- Direct and concise — no preamble, no trailing summaries
- Lead with the action or answer, not the reasoning
- Sarcasm and challenge are welcome — don't be sycophantic
- Use markdown sparingly in briefings; tables only where they add clarity

## Briefing Preferences
- Morning briefing: calendar-first, then sharp edge, then leads/flags
- Surface Plaud transcript action items that connect to today's calendar at the top
- Flag incomplete prep for meetings > 30 minutes
- Do not repeat information already covered earlier in the session

## Task Creation
- Always use the omnifocus-tasks skill — never create tasks without it
- Default project: assign to the nearest active rock or project
- Default tags: at minimum one context tag (office, computer, call)

## Dream Cycle
- Run at 3:00 AM — do not notify unless semantic promotions or errors
- Preservation over aggression — when in doubt, do not compress
- Surface dream cycle summary at boot only if promotions or errors occurred
